      Injects a Backstage token header when the user is signed in.

      Parameters

      • options: {
            allowUrl?: (url: string) => boolean;
            config?: Config;
            header?: { name: string; value: (backstageToken: string) => string };
            identityApi: IdentityApi;
            urlPrefixAllowlist?: string[];
        }

      Returns FetchMiddleware

      Remarks

      Per default, an Authorization: Bearer <token> is generated. This can be customized using the header option.

      The header injection only happens on allowlisted URLs. Per default, if the config option is passed in, the backend.baseUrl is allowlisted, unless the urlPrefixAllowlist or allowUrl options are passed in, in which case they take precedence. If you pass in neither config nor an allowlist/callback, the middleware will have no effect since effectively no request will match the (nonexistent) rules.

    StaticresolvePluginProtocol

    • resolvePluginProtocol(options: { discoveryApi: DiscoveryApi }): FetchMiddleware

      Handles translation from plugin:// URLs to concrete http(s) URLs based on the discovery API.

      Parameters

      • options: { discoveryApi: DiscoveryApi }

      Returns FetchMiddleware

      Remarks

      If the request is for plugin://catalog/entities?filter=x=y, the discovery API will be queried for 'catalog'. If it returned https://backstage.example.net/api/catalog, the resulting query would be https://backstage.example.net/api/catalog/entities?filter=x=y.

      If the incoming URL protocol was not plugin, the request is just passed through verbatim to the underlying implementation.
