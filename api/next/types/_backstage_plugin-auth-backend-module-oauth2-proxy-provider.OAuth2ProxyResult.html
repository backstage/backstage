    headers

    headers: IncomingHttpHeaders

    The headers of the incoming request from the OAuth2 proxy. This will include both the headers set by the client as well as the ones added by the OAuth2 proxy. You should only trust the headers that are injected by the OAuth2 proxy.

    Useful headers to use to complete the sign-in are for example x-forwarded-user and x-forwarded-email. See the OAuth2 proxy documentation for more information about the available headers and how to enable them. In particular it is possible to forward access and identity tokens, which can be user for additional verification and lookups.
