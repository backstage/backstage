    Optionaletag

    etag?: string

    An ETag which can be provided to check whether a UrlReaderService.readTree response has changed from a previous execution.

    Remarks

    In the UrlReaderService.readTree response, an ETag is returned along with the tree blob. The ETag is a unique identifier of the tree blob, usually the commit SHA or ETag from the target.

    When an ETag is given as a request option, UrlReaderService.readTree will first compare the ETag against the ETag on the target branch. If they match, UrlReaderService.readTree will throw a @backstage/errors#NotModifiedError indicating that the response will not differ from the previous response which included this particular ETag. If they do not match, UrlReaderService.readTree will return the rest of the response along with a new ETag.

    • filter?(path: string, info?: { size: number }): boolean

      A filter that can be used to select which files should be included.

      Parameters

      • path: string
      • Optionalinfo: { size: number }

      Returns boolean

      Remarks

      The path passed to the filter function is the relative path from the URL that the file tree is fetched from, without any leading '/'.

      For example, given the URL https://github.com/my/repo/tree/master/my-dir, a file at https://github.com/my/repo/blob/master/my-dir/my-subdir/my-file.txt will be represented as my-subdir/my-file.txt

      If no filter is provided, all files are extracted.
